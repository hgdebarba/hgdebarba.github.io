This assignment is part of the Practical Machine Learning coursera course.

A dataset on weight lifting is provided, in which movement tracking devices (comprising accelerometer, gyroscope and magnetometer) have been attached to the belt, arm, forearm and the dumbbell weight of six subjects. They were instructed to perform the weightlifting task in 5 different manners, out of which one was correct (A), and 4 containned postural problems (B, C, D and E). More information is available in http://groupware.les.inf.puc-rio.br/har.

We first load the training data:

## load training dataset

Excluding irrelevant variables

The dataset has 160 columns, most of which are irrelevant for this project, most notably timestamps, windows, X and user_name are not relevant.

By runningthe code bellow, I have also constated that most of the columns had almost all of their data missing.

print (paste0(names(data), ": ", colSums(is.na(data[,1:ncol(data)]))/nrow(data)))

Roughly 98% to 100% of the data was missing for all variables starting with avg, stddev, var, min, max, amplitude, kurtosis and skewness. I remove them with the regular expression bellow:

## select relevant columns
valid.data <- data[,grep("skewness|kurtosis|amplitude|min|max|var_|stddev|avg_|raw_|user|cvtd|window|X", names(data), invert = T)]

Removing these, we are left with 53 column. Out of which we have the response variable “classe” and 52 predictors.

Choice of algorithm

Given that this is a classification problem (classe variable can only assume a limited range of values), I decided to use the decision tree based technique random forest. Random forest is an stochastic technique that builds a multitude of decision trees in the training phase, when predictiong cathegorical values, it uses the mode of the decision tree results. It offers great accuracy, and is less suceptible to overfitting than a single decision tree https://en.wikipedia.org/wiki/Random_forest.

Random forest makes use of bootstrap to randomly pool from samples and variables. However, as a consequence of its complexity, it is a considerably slow classifier, for the provided dataset, it takes around 1 hours to train the model. I have used the “train” function present in the “caret” package to run the classifier. Cross validation with 10 folds was used, the training code is as follows:

require(caret)
set.seed(343876)
trainIndex = createDataPartition(valid.data$classe, p = 0.05,list=FALSE)
training = valid.data[trainIndex,]
set.seed(343876)
modFit <- train(classe ~ ., data = training, method = "rf", trControl = trainControl(method = "cv", number=10))
